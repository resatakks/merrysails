No, mixed keyword data is not a problem for analysis. It becomes a problem only if we mix languages on the same English commercial page.

Rules:

- Keep MerrySails public SEO direction `English-first` unless the user explicitly opens Turkish SEO.
- Do not insert Turkish keywords into English H1/H2/H3 just because the CSV contains TR volume.
- Store TR clusters as future opportunity memory.

## H1-H2-H3 Rules From The Cluster

### Global Rules

- H1 is not decorative. It must state the owner intent of the page.
- Do not spend H1 on generic brand copy if the page is commercial.
- Do not repeat the same exact keyword in every H2.
- H2s should divide major sub-intents.
- H3s should answer supporting questions, package differences, routes, prices, inclusions, and comparisons.
- Never let a support page take the exact H1 of a protected owner page.
- Never put Turkish query phrasing in English H1/H2/H3.

## GEO / AI Visibility Refresh

Sources reviewed:

- [Google SEO Starter Guide](https://developers.google.com/search/docs/fundamentals/seo-starter-guide)
- [Google AI features and your website](https://developers.google.com/search/docs/appearance/ai-features)
- [Google title links documentation](https://developers.google.com/search/docs/appearance/title-link)
- [Google structured data introduction](https://developers.google.com/search/docs/appearance/structured-data/intro-structured-data)
- [OpenAI crawler documentation](https://platform.openai.com/docs/bots)
- [IndexNow documentation](https://www.indexnow.org/documentation)

Working conclusions:

- Google says AI Overviews / AI Mode do not require special AI-only schema or AI-specific files; the same foundational SEO best practices matter.
- Pages must be indexable, eligible to show snippets, internally discoverable, and have important content available as visible text.
- Structured data must match visible content.
- High-quality images/videos can support AI/search visibility when relevant.
- OpenAI separates search and training crawlers: `OAI-SearchBot` is the important crawler for ChatGPT Search visibility, while `GPTBot` is related to training use.
- MerrySails should continue allowing AI/search crawlers while keeping content accurate and visible.
- IndexNow remains useful for Bing/Yandex discovery and should remain part of the indexing workflow.

## What This Means For Next Work

Before writing new content:

1. Finish H1/H2/H3 owner alignment on existing high-value pages.
2. Pull GSC query-to-page data for `24h`, `7d`, `28d`, and `3 months`.
3. Compare GSC query ownership against this cluster memory.
4. Only then adjust title/meta/internal links.
5. Only create new content if a validated query cluster has no clean owner.

Do not create new pages just because a keyword has volume. Create or edit pages only when the query has:

- clear commercial value
- a unique owner URL
- no dangerous cannibalization
- reliable offer truth
- evidence from GSC or Semrush that the page can serve the query better than the current page
